User:Sford > Sford wiki > Github tips

  • Create local project directory with LICENSE and README.md files.
  • Log into https://github.com using your account.
  • Click "+ New repository". Enter repo name and brief description. Do not create any initial files (like README). This should display a set of git commands, which can be modified appropriately. When I uploaded "blink", I did the following (from my Mac): 
git init
git add *
git commit -m "initial commit"
git remote add origin https://github.com/fordsfords/blink.git
git push -u origin master
  • Back on the web site, reload the project page. You should see your README.
  • Click "Branch: master" button, enter "gh-pages", and hit return.
  • Wait a minute or two. Or three. Then bring up https://github.com/fordsfords/blink/tree/gh-pages (with your project name instead of "blink").
  • Start up github GUI program.
  • File -> Add Local Repository...
- Browse to directory. Click "Add".


Update

  • Use github GUI program to ensure "master" branch is active. Do a "sync" (upper right-hand).
  • Edit a file
  • From github GUI program, enter change summary.
  • Click "Commit and Sync master"


Release stable

  • Start github GUI program and sync (upper right-hand).
  • Make sure you are on "gh-pages" branch.
  • Click "branches".
  • Select "Merge View".
  • Drag "master" to left box, "dh-pages" to right box. Should say, "Merging master into gh-pages".
  • Click "Merge Branches".
  • Click "Changes".
  • Click "sync" (upper right-hand).
  • Switch back to "master" branch.
